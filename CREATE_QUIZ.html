Danh sách câu hỏi

Hướng dẫn sử dụng

Định dạng xuất: Công cụ tạo file .xml theo chuẩn E-VLUTE XML (định dạng XML) — Import vào hệ thống Elearning-VLUTE qua Ngân hàng câu hỏi → Nhập → định dạng XML. Đây là định dạng gói được cả 6 loại câu hỏi kèm hình ảnh (ảnh trong câu và ảnh trong đáp án).

Các loại câu hỏi

  • Trắc nghiệm: một hoặc nhiều đáp án đúng, mỗi phương án gắn được ảnh riêng.
  • Đúng / Sai: một phát biểu, chọn đáp án đúng là Đúng hay Sai.
  • Trả lời ngắn: người học gõ đáp án, so khớp văn bản.
  • Điền khuyết: đặt đáp án cần điền trong dấu [ ] ngay trong câu. VD: Thủ đô Việt Nam là [Hà Nội]. — nhiều đáp án đúng cho một chỗ: [Hà Nội|Ha Noi].
  • Nối cặp: ghép vế trái với vế phải tương ứng.

Dữ liệu của bạn được lưu thế nào?

  • Tự động lưu trên trình duyệt: mỗi thay đổi lưu ngay vào máy — mở lại trang là còn nguyên (chỉ chạy khi mở file trên trình duyệt của bạn).
  • Lưu ra file .json: Dự án → Lưu dự án ra file để mang sang máy khác; mở lại bằng Dự án → Mở dự án từ file. File này chứa cả ảnh.
  • Xoá toàn bộ: Dự án → Xoá toàn bộ dữ liệu (có xác nhận).

Nhập từ Excel

Tải mẫu, điền theo cột rồi nhập lại. Cột Loại: tracnghiem, dung_sai, tra_loi_ngan, dien_khuyet, noi_cap.

  • tracnghiem: phương án ở cột A–F; cột Đáp án đúng ghi A hoặc A,C.
  • dung_sai: cột Đáp án đúng ghi Đúng hoặc Sai.
  • tra_loi_ngan: mỗi ô A–F là một đáp án được chấp nhận.
  • dien_khuyet: viết câu ở cột "Nội dung", đặt đáp án trong [ ].
  • noi_cap: mỗi ô A–F ghi Vế trái = Vế phải.

Nhập từ Word (.docx)

Dùng cho trắc nghiệm đánh số, đặt dấu * sau phương án đúng, mỗi câu cách nhau một dòng trống:

Câu 1: Thủ đô của Việt Nam là?
A. Hà Nội*
B. TP. Hồ Chí Minh
C. Đà Nẵng
D. Huế

Cũng chấp nhận dòng ĐÁP ÁN: B thay cho dấu *.

Nạp vào E-VLUTE

  • Bấm Xuất E-VLUTE → tải file .xml.
  • Vào Ngân hàng câu hỏi → Nhập → chọn định dạng XML → tải file lên → Nhập.
Câu Nối cặp: cần tối thiểu 2 cặp đủ 2 vế và ≥3 vế phải. Nên tạo từ 3 cặp trở lên.
